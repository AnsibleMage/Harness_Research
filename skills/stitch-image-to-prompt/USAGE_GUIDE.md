# stitch-image-to-prompt 사용법 가이드

> 이미지를 Google Stitch UI 디자인으로 변환하는 스킬의 실전 사용법

---

## 1. 전제 조건

| 항목 | 필요 여부 | 설명 |
|------|-----------|------|
| Google Stitch MCP 연결 | **필수** | `settings.json`에 stitch MCP 서버 설정 완료 상태여야 함 |
| Google 계정 로그인 | **필수** | stitch.withgoogle.com에 로그인된 상태 |
| 이미지 파일 | **필수** | 스크린샷, 디자인 시안, 와이어프레임 등 |
| 크롬에서 Stitch 열기 | 권장 | 생성 결과를 실시간 확인하기 위함 |

### MCP 연결 확인 방법

Claude Code에서 아래처럼 테스트:
```
"스티치 프로젝트 목록 보여줘"
→ mcp__stitch__list_projects 호출
→ 프로젝트 목록이 나오면 정상
```

## 2. 기본 사용법

### Step 1: 이미지 준비

지원하는 이미지 소스:

| 소스 | 예시 | 팁 |
|------|------|-----|
| 웹 스크린샷 | 브라우저 전체 캡처 | 주소표시줄 제외하고 캡처하면 깔끔 |
| Figma 캡처 | 프레임 선택 → Export | 1x 해상도면 충분 |
| 손그림 와이어프레임 | 종이에 그린 걸 사진 촬영 | 밝은 곳에서 촬영, 글씨 선명하게 |
| 앱 스크린샷 | 모바일/데스크톱 앱 캡처 | 상태바 포함/제외 선택 |
| 디자인 시안 | PSD/AI에서 Export한 이미지 | PNG 또는 JPG 권장 |

### Step 2: Claude Code에 요청

```
[이미지 첨부] + "스티치로 만들어줘"
```

또는 더 구체적으로:
```
[이미지 첨부] + "이 대시보드를 스티치 프롬프트로 변환해줘. 데스크톱용이고 다크모드야"
```

### Step 3: 인터뷰 응답

아리가 2-3개 질문을 해. 간단히 답하면 돼:

```
아리: "어떤 용도인가요? 디바이스는?"
앤:  "프로토타이핑용, 데스크톱"

아리: "픽셀퍼펙트 재현? 아니면 구조만?"
앤:  "구조만 잡아줘"
```

> **빠른 진행**: "빨리 해줘" 또는 "디폴트로 진행" 이라고 하면 인터뷰를 건너뛰고 바로 생성

### Step 4: 프롬프트 확인 & 실행

아리가 영어 프롬프트를 보여주고 3가지 옵션을 제시:

```
1) 복사해서 Stitch 웹에 직접 붙여넣기
2) 새 프로젝트 생성 + 스크린 자동 생성
3) 기존 프로젝트에 스크린 추가
```

**추천**: 옵션 2 또는 3 선택 → MCP로 자동 실행

### Step 5: 결과 확인 & 수정

크롬에서 스티치보드 확인 후, 수정 요청:

```
"헤더 배경색을 네이비로 바꿔줘"
"카드 레이아웃을 3열에서 2열로 변경해줘"
"다른 색상 버전도 3개 만들어줘"
```

## 3. 실전 시나리오별 가이드

### 시나리오 A: 웹사이트 스크린샷 → Stitch 재현

```
[랜딩페이지 스크린샷 첨부]
"이 랜딩페이지를 스티치로 만들어줘. 텍스트는 한국어 그대로 유지해줘"
```

### 시나리오 B: 손그림 와이어프레임 → UI 프로토타입

```
[손그림 사진 첨부]
"이 와이어프레임을 깔끔한 UI로 만들어줘. 모바일 앱이고 밝은 느낌으로"
```

### 시나리오 C: Figma 디자인 → Stitch로 변형 탐색

```
[Figma 캡처 첨부]
"이 디자인 기반으로 스티치에서 레이아웃 변형 5개 만들어줘"
```

→ 이 경우 `generate_variants` 도구 사용 (레이아웃/색상/폰트 변형)

### 시나리오 D: 경쟁사 앱 분석 → 우리 버전 프로토타이핑

```
[경쟁사 앱 스크린샷 첨부]
"이 구조를 참고해서 우리 브랜드 색상(#1E40AF)으로 스티치 디자인 만들어줘"
```

### 시나리오 E: 기존 Stitch 프로젝트에 화면 추가

```
[새 화면 디자인 이미지 첨부]
"프로젝트 2119859971848534106에 이 화면 추가해줘"
```

## 4. 실행 후 활용 명령어

스크린이 생성된 뒤 바로 사용할 수 있는 후속 명령어:

### 수정 (edit_screens)

```
"헤더의 로고를 왼쪽 정렬해줘"
"CTA 버튼을 더 크게, 빨간색으로 바꿔줘"
"사이드바 메뉴에 '설정' 항목 추가해줘"
"모든 카드에 그림자 효과 넣어줘"
```

> 한 번에 하나의 변경만 요청하는 것이 품질이 좋음

### 변형 생성 (generate_variants)

```
"레이아웃 변형 3개 보여줘"           → aspects: [LAYOUT]
"다른 색상 조합으로 5개 만들어줘"     → aspects: [COLOR_SCHEME]
"완전히 다른 느낌으로 재해석해줘"     → creativeRange: REIMAGINE
"미세하게 다듬은 버전 3개"           → creativeRange: REFINE
"폰트만 바꾼 버전들 보여줘"          → aspects: [TEXT_FONT]
```

### 조회

```
"이 프로젝트의 모든 스크린 보여줘"    → list_screens
"프로젝트 목록 보여줘"               → list_projects
"스크린 상세 정보 보여줘"             → get_screen
```

## 5. 최적의 결과를 위한 팁

### DO (이렇게 해)

| # | 팁 | 예시 |
|---|-----|------|
| 1 | 이미지는 선명하게 | 흐릿한 스크린샷보다 깨끗한 캡처 |
| 2 | 원하는 부분만 크롭 | 전체 화면보다 특정 섹션 캡처가 정밀도 높음 |
| 3 | 브랜드 색상이 있으면 알려줘 | "기본 색상은 #2563EB야" |
| 4 | 디바이스를 명시해 | "모바일용" / "데스크톱용" |
| 5 | 수정은 하나씩 | "색상 바꾸고 폰트도 바꾸고 레이아웃도" → 3번에 나눠서 |
| 6 | Vibe 형용사 활용 | "미니멀하고 프로페셔널한 느낌" |

### DON'T (이건 피해)

| # | 안티패턴 | 이유 |
|---|---------|------|
| 1 | CSS/코드 문법으로 요청 | Stitch는 자연어만 이해 |
| 2 | 한 프롬프트에 여러 화면 | 화면 하나씩이 품질 좋음 |
| 3 | "좀 더 좋게 해줘" | 모호함 → "버튼을 더 크게"처럼 구체적으로 |
| 4 | 파일명으로 이미지 지칭 | "logo.png 써줘" → "좌상단에 회사 로고" |
| 5 | 한국어 프롬프트 | Stitch 엔진은 영어가 훨씬 정확 (변환은 아리가 해줌) |

## 6. 모델 선택 가이드

| 상황 | 추천 모델 | 이유 |
|------|----------|------|
| 복잡한 대시보드, 많은 컴포넌트 | GEMINI_3_1_PRO | 최고 품질, 디테일 정확 |
| 빠른 프로토타이핑, 단순 화면 | GEMINI_3_FLASH | 속도 우선, 반복에 유리 |
| 처음 만들 때 | GEMINI_3_1_PRO | 첫 결과가 좋아야 수정이 적음 |
| 변형 탐색할 때 | GEMINI_3_FLASH | 많이 만들어보고 고르는 용도 |

> 기본값: GEMINI_3_1_PRO (별도 지정 안 하면)

## 7. 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| "MCP 도구를 찾을 수 없습니다" | Stitch MCP 미연결 | `settings.json`에서 stitch MCP 설정 확인 |
| 생성이 오래 걸림 (2-3분) | 정상. Stitch AI 처리 시간 | 기다려. 재시도하지 말 것 |
| 연결 오류 후 결과 없음 | 서버 측 생성은 진행 중일 수 있음 | `list_screens`로 확인 |
| 결과가 이미지와 많이 다름 | 프롬프트가 너무 추상적 | 더 구체적인 프롬프트로 재생성 |
| 색상이 의도와 다름 | HEX 값 미지정 | 정확한 색상값을 프롬프트에 포함 |
| 한국어 텍스트가 깨짐 | 프롬프트에 한국어 직접 포함 | 텍스트는 프롬프트에 그대로 넣으면 됨 |

## 8. 스킬 파일 구조

```
~/.claude/skills/stitch-image-to-prompt/
├── SKILL.md                           ← 스킬 본체 (Phase 0~4)
├── USAGE_GUIDE.md                     ← 이 문서 (사용법 가이드)
└── references/
    └── stitch-prompting-guide.md      ← 상세 프롬프팅 레퍼런스
```

| 파일 | 용도 | 읽는 시점 |
|------|------|----------|
| SKILL.md | 스킬 트리거 시 자동 로드 | 매번 (스킬 실행 시) |
| USAGE_GUIDE.md | 사용자 참조용 | 사용법 궁금할 때 |
| stitch-prompting-guide.md | 프롬프트 작성 중 참조 | 디테일 확인 필요 시 |
