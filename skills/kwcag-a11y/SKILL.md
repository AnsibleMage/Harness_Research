# KWCAG 2.2 웹 접근성 검사 스킬

## 1. 개요

KWCAG 2.2(한국형 웹 콘텐츠 접근성 지침 2.2)의 33개 검사항목을 기반으로 HTML 파일의 접근성을 검사한다.
3단계 검사 체계로 자동화 가능한 항목은 스크립트로, 불가능한 항목은 수동 체크리스트로 제공한다.

**KWCAG 2.2 구조**: 4원칙(인식·운용·이해·견고) → 14지침 → 33검사항목

## 2. 실행 흐름

```
사용자 요청 → 대상 파일/디렉토리 확인
    ├─ 단일 파일 → Tier 1 실행 → (선택) Tier 2 → 보고서 생성
    ├─ 프로젝트 전체 → Glob **/*.html → 파일별 Tier 1 → 집계 → 보고서
    └─ 체크리스트만 → Tier 3 수동 검토 체크리스트 생성
```

### 호출 방법
- `/kwcag-a11y` 또는 `/kwcag-a11y <파일경로>`
- 자연어: "이 HTML 접근성 검사해줘", "KWCAG 기준으로 체크해줘"

### 스캔 모드 결정

| 사용자 요청 | 모드 | 동작 |
|------------|------|------|
| "간단히 체크" / 빠른 검사 | Tier 1만 | Python 정적 분석 (의존성 없음) |
| "자세히 검사" / 전체 검사 | Tier 1 + 2 | 정적 분석 + axe-core DOM 분석 |
| "체크리스트만" | Tier 3만 | 수동 검토 체크리스트 생성 |
| 기본 (모드 미지정) | Tier 1 + 3 | 정적 분석 + 수동 체크리스트 |

## 3. Tier 1 — 정적 분석 (Python, 의존성 없음)

Python 표준 라이브러리 `html.parser`로 HTML을 파싱하여 18/33 검사항목을 검사한다.
외부 패키지 설치 없이 즉시 실행 가능하여 기본 검사 도구로 사용한다.

### 실행 방법

```bash
python ~/.claude/skills/kwcag-a11y/scripts/kwcag-static-check.py <파일_또는_디렉토리> [옵션]
```

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--format` | 출력 형식 (`json`, `md`) | `json` |
| `--severity` | 최소 심각도 (`critical`, `major`, `minor`, `all`) | `all` |

### 커버리지 (18/33)

| 원칙 | 커버 항목 |
|------|----------|
| 인식 (5/9) | 5.1.1 대체텍스트, 5.2.1 자막, 5.3.1 표구성, 5.3.2 선형구조, 5.4.2 자동재생 |
| 운용 (7/15) | 6.1.1 키보드, 6.1.2 초점, 6.1.4 단축키, 6.4.1 건너뛰기, 6.4.2 제목, 6.4.3 링크텍스트, 6.5.3 레이블네임 |
| 이해 (4/7) | 7.1.1 언어표시, 7.2.1 자동실행, 7.3.2 레이블, 7.3.3 인증 |
| 견고 (2/2) | 8.1.1 마크업오류, 8.2.1 웹앱접근성 |

Tier 1에서 **미커버** (Tier 2 또는 수동 필요): 5.3.3 지시사항, 5.4.1 색독립성, 5.4.3 명도대비, 5.4.4 콘텐츠구분, 6.1.3 조작가능, 6.2.1 응답시간, 6.2.2 정지기능, 6.3.1 깜빡임, 6.4.4 참조위치, 6.5.1 단일포인터, 6.5.2 포인터취소, 6.5.4 동작기반, 7.2.2 도움정보, 7.3.1 오류정정, 7.3.4 반복입력

## 4. Tier 2 — axe-core 강화 분석 (Node.js)

axe-core + jsdom으로 DOM 기반 접근성 분석을 수행한다. 브라우저 없이 동작한다.
특히 **명도 대비(5.4.3)** 자동 계산이 가능하여 Tier 1을 보완한다.

### 사전 설치 (최초 1회)

```bash
cd ~/.claude/skills/kwcag-a11y && npm init -y && npm install axe-core jsdom
```

### 실행 방법

```bash
node ~/.claude/skills/kwcag-a11y/scripts/kwcag-axe-check.js <파일_또는_디렉토리> [옵션]
```

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--format` | 출력 형식 (`json`, `md`) | `json` |
| `--severity` | 최소 심각도 | `all` |

### 완전 커버 항목 (17/33)

5.1.1, 5.3.1, 5.3.2, 5.4.3(명도대비), 6.1.1, 6.1.2, 6.4.1, 6.4.2, 6.4.3, 6.5.3,
7.1.1, 7.3.2, 7.3.4, 8.1.1, 8.2.1 + 부분 커버 다수

axe-core 위반 결과는 KWCAG 요구사항 번호로 자동 매핑된다.
매핑 테이블: `references/axe-core-mapping.md` 참조.

자동화 불가능한 9개 항목에 대해 구조화된 한국어 체크리스트를 생성한다.

## 6. 통합 보고서 생성

Tier 1, 2, 3의 결과를 병합하여 한국어 종합 보고서 2종을 생성한다.
사용자가 유형을 지정하지 않으면 기본으로 둘 다 생성한다.

### 2종 보고서

| 구분 | 용도 | 내용 |
|------|------|------|
| 01 요약보고서 | 관리자/PM용 | 준수율, 심각도 현황, 수정 우선순위 |
| 02 디테일보고서 | 개발자용 | 파일별 전수 위반 + 오류코드/정상코드 예시 |

### 파일명 규칙

보고서 파일명은 아래 규칙을 따른다. 프로젝트명과 날짜로 2종을 묶어 식별하기 쉽게 한다.

```
01_[프로젝트명]_웹접근성_요약보고서_[YYYYMMDD].md
02_[프로젝트명]_웹접근성_디테일보고서_[YYYYMMDD].md
```

**예시**:
```
01_gil-seoul_fo_웹접근성_요약보고서_20260320.md
02_gil-seoul_fo_웹접근성_디테일보고서_20260320.md
```

**규칙**:
- `01_` = 요약 보고서 (먼저 읽을 것)
- `02_` = 디테일 보고서 (수정 작업 시 참조)
- 프로젝트명은 대상 폴더명 또는 사용자 지정 이름
- 날짜는 점검일 기준 YYYYMMDD

### 실행 방법

```bash
python ~/.claude/skills/kwcag-a11y/scripts/kwcag-report-gen.py \
  --tier1 tier1_result.json \
  --tier2 tier2_result.json \
  --output report.md \
  --format md
```

### 요약보고서 구조 (01)

```
# KWCAG 2.2 웹 접근성 검사 보고서
## 검사 개요 — 대상, 일시, 도구, 범위
## 요약 — 원칙별 준수율 테이블 + 심각도별 현황
## 상세 결과 — 원칙 → 검사항목 → 위반건별 (위치, 요소, 문제, 수정 권고)
## 수동 검토 체크리스트 — 자동화 불가 항목 체크박스 목록
## 수정 우선순위 — P1 즉시(Critical) / P2 조기(Major) / P3 개선(Minor)
```

### 디테일보고서 구조 (02)

```
# 웹 접근성 상세 수정 가이드
## 공통 레이아웃 일괄 수정 (최우선)
## 파일별 상세 수정 가이드
  ### 📄 파일경로
    #### 🔴 #N [검사항목] — 오류코드 + 정상코드 예시
```

## 7. 심각도 분류

| 등급 | 한국어 | 기준 | 예시 |
|------|--------|------|------|
| Critical | 심각 | 콘텐츠 접근 불가 | alt 누락, 키보드 접근 불가 |
| Major | 주요 | 접근에 상당한 어려움 | 명도대비 부족, 레이블 누락 |
| Minor | 경미 | 접근에 약간의 불편 | 제목 수준 건너뛰기, 빈 링크 |
| Info | 참고 | 검토 필요 | 장식 이미지 확인, accesskey 사용 |

상세 기준: `references/severity-levels.md`

## 8. Claude Code 통합 가이드

### 실행 워크플로우

1. **대상 확인**: 사용자가 지정한 파일/디렉토리 확인. 미지정 시 현재 디렉토리의 `**/*.html`
2. **Tier 1 실행**: `python kwcag-static-check.py <대상> --format json` → JSON 결과 파싱
3. **(선택) Tier 2 실행**: npm 의존성 확인 후 `node kwcag-axe-check.js <대상> --format json`
4. **보고서 생성**: `python kwcag-report-gen.py` 또는 Claude가 직접 JSON을 읽어 한국어 보고서 작성
5. **수정 제안**: 위반 항목별 구체적 코드 수정안 제시
6. **수동 체크리스트**: Tier 3 체크리스트를 보고서 말미에 첨부

### 수정 적용 워크플로우

위반 발견 후 사용자가 수정을 요청하면:
1. 심각도 Critical → Major → Minor 순으로 수정
2. 각 수정 사항에 KWCAG 요구사항 번호 주석 추가
3. 수정 후 재검사하여 해결 확인

### Tier 2 사용 판단

| 조건 | Tier 2 실행 여부 |
|------|----------------|
| `node_modules/axe-core` 존재 | 실행 |
| npm 사용 가능 + 사용자 동의 | 설치 후 실행 |
| npm 없음 또는 거부 | Tier 1만으로 진행 |

## 9. 제한 사항

- **CSS 기반 명도대비**: Tier 1은 인라인 스타일만 검사. 외부 CSS 명도대비는 Tier 2 필요
- **동적 콘텐츠**: JavaScript로 생성되는 DOM은 Tier 1에서 검사 불가
- **배경 이미지 위 텍스트**: 자동 분석 불가, 수동 검토 필요
- **멀티페이지 일관성**: 페이지 간 네비게이션 일관성은 수동 확인 필요
- **모바일 특화 항목**: 터치 제스처, 모션 센서는 수동 검토만 가능
