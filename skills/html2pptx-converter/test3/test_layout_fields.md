0. 사용자 인터페이스 구성 요소

항목명: User Interface Elements
역할 설명: 페이지를 구성하는 모든 시각적 요소 및 상호작용 요소들
표시 위치: 화면 전체에 분산 배치
입력 형식: 클릭, 호버, 링크 이동 등 기본적인 웹 상호작용
필수 여부: ✅ 레이아웃 검증에 필수적인 요소들
기능 범위: 좌표 검증, 레이아웃 검증, 도형 렌더링 검증, 텍스트 표시 검증

- 색상: 빨강 계열 배경, 흰색 텍스트

- PPTX 변환 후 → 슬라이드 좌상단 (0, 0) 좌표에 정확히 위치하는지 확인

12. 게시판 테이블

항목명: Notice Board Data Table
역할 설명: 공지사항을 테이블 형식으로 표시하며 카테고리별 컬러 태그 적용
표시 위치: 메인 콘텐츠 좌측 (2단 레이아웃, flex:2 비율)
입력 형식: 게시물 클릭, 카테고리 필터
필수 여부: ✅ 테이블 구조 및 태그 색상 검증 핵심 요소
테이블 구조:
- 헤더: "Notice Board" 제목 + "5 New" 빨간 뱃지
- 열 구성: No, Category, Title, Date, Views (5열)
- 행 데이터: 5건의 게시물

게시물 데이터:
- #125: Notice — System maintenance scheduled (2025-03-10, 1,842 views)
- #124: Event — Developer conference 2025 (2025-03-08, 956 views)
- #123: Update — Platform v3.2 release notes (2025-03-05, 2,103 views)
- #122: FAQ — How to reset your password (2025-03-01, 3,241 views)
- #121: Notice — Privacy policy update (2025-02-28, 1,567 views)

카테고리 태그:
- Notice: 빨강 계열 (연한 빨강 배경, 진한 빨강 텍스트)
- Event: 초록 계열 (연한 초록 배경, 진한 초록 텍스트)
- Update: 파랑 계열 (연한 파랑 배경, 진한 파랑 텍스트)
- FAQ: 노랑 계열 (연한 노랑 배경, 진한 갈색 텍스트)

연결 흐름
- 게시물 제목 클릭 → 상세 페이지로 이동
- 카테고리 태그 클릭 → 해당 카테고리 필터링
- PPTX 변환 시 → 테이블 열/행 정렬, 태그 색상 구분이 정확한지 확인

15. 도형 테스트 5종

항목명: Shape Rendering Test Elements
역할 설명: PPTX 도형 변환 정확도를 검증하기 위한 5종 도형
표시 위치: 도형 & 라인 테스트 영역 상단, 수평 배치
입력 형식: 읽기 전용 (시각적 검증 요소)
필수 여부: ✅ 도형 렌더링 검증 핵심 요소
도형 구성:
- Rectangle — 파란 배경, 약간의 라운드 모서리
- Rounded Rect — 초록 배경, 큰 라운드 모서리
- Circle — 빨간 배경, 완전한 원형
- Border Only — 배경 없음, 금색 테두리만 표시
- Box Shadow — 보라 배경, 그림자 효과 적용
- 각 도형 아래에 이름 라벨 텍스트

연결 흐름
- PPTX 변환 시 → 각 도형의 모양, 색상, 라운드 값이 정확히 재현되는지 확인
- Border Only → 배경 없이 테두리만 렌더링되는지 확인
- Box Shadow → 그림자 효과가 표현되는지 확인 (PPTX 제한 사항 있을 수 있음)

- Top Border — 빨간색, 상단에만 두꺼운 테두리
- Left Border — 파란색, 좌측에만 두꺼운 테두리
- Bottom Border — 초록색, 하단에만 두꺼운 테두리
- Full Border — 금색, 전체 테두리 + 라운드 모서리
- 각 라인 요소 아래에 이름 라벨 텍스트

연결 흐름
- PPTX 변환 시 → 부분 테두리가 지정된 방향(상/좌/하)에만 표시되는지 확인
- Full Border → 전체 테두리가 균일하게 표시되는지 확인
- 각 라인의 색상과 두께가 정확히 변환되는지 확인
