# TestCorp Portal - 웹 포탈 테스트 페이지 기능 설명

---

1. 전체 화면 구성

화면 레이아웃
- 최상단: 코너 마커 (좌표 검증용, 네 꼭짓점 + 정중앙)
- 상단: 헤더 네비게이션 (로고 + 메뉴)
- 중앙 상단: 히어로 배너 (제목 + 설명 + CTA 버튼)
- 중앙: 통계 요약 바 (핵심 수치 4개)
- 중앙 하단: 게시판(좌측) + 사이드바(우측) 2단 레이아웃
- 하단: 도형 & 라인 렌더링 테스트 영역
- 최하단: 푸터 (4단 그리드 + 저작권)

---

1-1. 코너 마커
위치: 슬라이드 네 꼭짓점 및 정중앙 (페이지 전체 영역 위에 오버레이)

기능 설명
- 목적: PPTX 변환 시 좌표 매핑 정확도를 검증하기 위한 기준점
- 구성: 5개 마커 (TL, TR, BL, BR, CENTER)
- 각 마커는 고유한 색상과 라벨 텍스트를 가짐
- 네 꼭짓점 마커는 정사각형, 중앙 마커는 원형으로 구분
- 각 꼭짓점 마커는 해당 방향의 모서리만 라운드 처리 (TL은 우하단만, TR은 좌하단만 등)
- PPTX 슬라이드에서 경계와 중심점에 정확히 배치되어야 함

연결 흐름
- 마커 위치 확인 → 변환 후 슬라이드 좌표와 1:1 매핑 여부 검증
- 중앙 마커 → 슬라이드 정중앙(50%, 50%) 정렬 정확도 확인
- 코너 마커 간 거리 → 슬라이드 전체 크기 스케일링 검증

---

1-2. 헤더 네비게이션
위치: 페이지 최상단 고정 영역

기능 설명
- 좌측: "TestCorp Portal" 브랜드 로고 텍스트
- 우측: 네비게이션 메뉴 링크 5개 수평 배치
  • Home (현재 페이지 - 하단 밑줄로 활성 표시)
  • Services
  • Board
  • About
  • Contact
- 어두운 네이비 배경 위에 밝은 텍스트 배치
- 하단에 파란색 구분선으로 본문 영역과 시각적 분리

연결 흐름
- 로고 클릭 → 메인 페이지로 이동
- 각 메뉴 항목 클릭 → 해당 기능 페이지로 이동
- 활성 메뉴(Home)는 시각적 피드백으로 현재 위치 인식 지원

---

1-3. 히어로 섹션
위치: 헤더 바로 아래, 페이지 중앙 상단 배너 영역

기능 설명
- 목적: 서비스 소개 및 사용자 행동 유도(CTA)를 위한 주요 배너
- 제목: "Welcome to TestCorp Portal" — 대형 텍스트로 시선 집중
- 부제목: html2pptx 변환 테스트를 위한 레이아웃 검증 페이지임을 안내하는 설명 텍스트
- 파란색 배경 위에 흰색 텍스트로 높은 대비 유지
- CTA 버튼 2개:
  • "Get Started" — 채움형 버튼 (빨간 배경, 흰 텍스트) — 주요 행동 유도
  • "Learn More" — 테두리형 버튼 (투명 배경, 흰 테두리) — 보조 행동 유도
- 제목, 부제목, 버튼 모두 수평 중앙 정렬

연결 흐름
- "Get Started" 클릭 → 서비스 등록/시작 페이지로 이동
- "Learn More" 클릭 → 서비스 상세 소개 페이지로 이동
- 히어로 영역은 스크롤 시 자연스럽게 본문 콘텐츠로 전환

---

1-4. 통계 요약 바
위치: 히어로 섹션 바로 아래, 본문 콘텐츠 시작 전

기능 설명
- 목적: 서비스의 핵심 성과 지표를 한눈에 전달
- 4개 수치 항목을 수평 균등 배치:
  • 1,247 / Active Users — 현재 활성 사용자 수
  • 358 / Projects — 진행 중인 프로젝트 수
  • 99.9% / Uptime — 서비스 가동률
  • 24/7 / Support — 연중무휴 지원 체제
- 큰 숫자(파란색, 굵은 글씨)와 작은 라벨(대문자, 회색)의 대비로 정보 위계 표현
- 흰색 배경에 하단 구분선으로 다음 섹션과 시각적 분리

연결 흐름
- 각 통계 항목 클릭 → 해당 지표의 상세 대시보드로 이동
- 수치는 실시간 데이터 연동으로 자동 갱신

---

1-5. 게시판
위치: 통계 바 아래, 메인 콘텐츠 좌측 영역 (2단 레이아웃의 왼쪽, 2/3 비율)

기능 설명
- 섹션 제목: "Notice Board" + "5 New" 빨간 뱃지로 신규 게시물 수 표시
- 데이터 테이블 구조: 5열 (No, Category, Title, Date, Views) × 5행
- 카테고리별 컬러 태그로 게시물 유형 시각 구분:
  • Notice (빨강 태그) — 공지사항
  • Event (초록 태그) — 이벤트/행사
  • Update (파랑 태그) — 업데이트/변경사항
  • FAQ (노랑 태그) — 자주 묻는 질문
- 게시물 목록:
  • #125: Notice — System maintenance scheduled (2025-03-10, 1,842 views)
  • #124: Event — Developer conference 2025 (2025-03-08, 956 views)
  • #123: Update — Platform v3.2 release notes (2025-03-05, 2,103 views)
  • #122: FAQ — How to reset your password (2025-03-01, 3,241 views)
  • #121: Notice — Privacy policy update (2025-02-28, 1,567 views)
- 행 사이 연한 구분선으로 가독성 확보

연결 흐름
- 게시물 제목 클릭 → 해당 게시물 상세 페이지로 이동
- 카테고리 태그 클릭 → 해당 카테고리 필터링 적용
- "5 New" 뱃지 → 최근 24시간 내 등록된 게시물 강조

---

1-6. 사이드바
위치: 통계 바 아래, 메인 콘텐츠 우측 영역 (2단 레이아웃의 오른쪽, 1/3 비율)

기능 설명
- 카드 1 — Recent Updates: 최근 시스템 업데이트 4건을 리스트 형태로 표시
  • API v2.5 released (03-10)
  • Dashboard redesign (03-08)
  • Mobile app update (03-05)
  • Security patch applied (03-01)
- 각 항목은 제목과 날짜로 구성, 항목 사이 구분선
- 카드 2 — Quick Links: 자주 사용하는 기능 4개를 2×2 그리드로 배치
  • Docs (📋) — 문서 페이지
  • Support (💬) — 고객 지원
  • Reports (📊) — 리포트 대시보드
  • Settings (⚙️) — 환경 설정
- 각 카드는 흰 배경, 둥근 모서리, 연한 테두리로 통일된 카드 스타일 적용

연결 흐름
- Recent Updates 각 항목 클릭 → 해당 업데이트 상세 내역 페이지로 이동
- Quick Links 각 항목 클릭 → 해당 기능 페이지로 바로 이동
- 카드 간 간격은 일정하게 유지하여 시각적 균형 확보

---

1-7. 도형 & 라인 테스트
위치: 게시판/사이드바 아래, 푸터 위 전용 테스트 영역

기능 설명
- 목적: PPTX 변환 엔진의 도형/라인 렌더링 정확도를 전용으로 검증
- 도형 5종:
  • Rectangle — 기본 직사각형 (약간의 라운드)
  • Rounded Rect — 둥근 모서리 사각형 (큰 라운드)
  • Circle — 완전한 원형
  • Border Only — 배경 없이 테두리만 있는 사각형
  • Box Shadow — 그림자 효과가 적용된 사각형
- 라인(부분 테두리) 4종:
  • Top Border — 상단에만 테두리
  • Left Border — 좌측에만 테두리
  • Bottom Border — 하단에만 테두리
  • Full Border — 전체 테두리 + 라운드 모서리
- 각 도형/라인 아래에 이름 라벨 텍스트 표시

연결 흐름
- 각 도형의 모양, 색상, 라운드, 그림자가 PPTX에서 정확히 재현되는지 확인
- 부분 테두리가 올바른 방향으로만 표시되는지 확인
- Border Only는 배경 없이 테두리만 렌더링되는지 확인

---

1-8. 푸터
위치: 페이지 최하단

기능 설명
- 4단 그리드 레이아웃으로 구성된 서비스 정보 영역
- 컬럼 1 (넓은 폭): TestCorp 회사 소개 텍스트
- 컬럼 2: Company 카테고리 — About Us, Careers, Blog, Press
- 컬럼 3: Resources 카테고리 — Documentation, Help Center, API Status, Downloads
- 컬럼 4: Legal 카테고리 — Privacy Policy, Terms of Service, Cookie Policy, GDPR
- 어두운 배경 위에 밝은 회색 텍스트로 가독성 확보
- 하단 바: 구분선 아래에 저작권 텍스트(좌측)와 소셜 링크 3개(우측, GitHub/Twitter/LinkedIn)

연결 흐름
- 각 링크 클릭 → 해당 정보 페이지로 이동
- 소셜 링크 클릭 → 외부 소셜 미디어 페이지로 새 탭 열기
- 각 카테고리 제목은 해당 분야의 페이지 목록을 그룹핑
