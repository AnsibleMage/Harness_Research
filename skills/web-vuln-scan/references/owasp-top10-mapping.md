# OWASP Top 10 (2021) — 행안부/KISA 역매핑

## 개요
OWASP Top 10 (2021) 각 카테고리에서 대응하는 행안부 보안약점(MOIS-ID)과 KISA 웹취약점(WV-ID)을 역으로 매핑한 참조 문서.

---

## 매핑 테이블

|------------|----------|-----------|----------|-----------|
| A01 | Broken Access Control | IV-02,06,10 SF-02,03 | WV-17,25 | 3/5 |
| A02 | Cryptographic Failures | SF-04,05,06,08 | WV-27 | 4/4 |
| A03 | Injection | IV-01,03,04,07,08,09,11,12,14,15 | WV-01~07,11 | 7/10 |

## 카테고리별 상세

### A01:2021 Broken Access Control (접근 제어 취약)
- **설명**: 사용자가 허용된 범위를 벗어나 행동할 수 있는 취약점
- **대응 MOIS**: IV-02(경로조작), IV-06(URL리다이렉트), IV-10(CSRF), SF-02(부적절한 인가), SF-03(권한설정)
- **대응 KISA**: WV-17(불충분한 인가), WV-25(경로 추적)
- **공공기관 주의**: 관리자 페이지 접근 제어, 타 사용자 정보 열람 방지 필수

### A02:2021 Cryptographic Failures (암호화 실패)
- **설명**: 민감 데이터 보호를 위한 암호화가 부재하거나 취약
- **대응 MOIS**: SF-04(취약 알고리즘), SF-05(평문저장), SF-06(평문전송), SF-08(짧은 키)
- **대응 KISA**: WV-27(데이터 평문 전송)
- **공공기관 주의**: 개인정보보호법 안전성 확보조치 기준 준수, 국정원 인증 암호 모듈 사용

### A03:2021 Injection (인젝션)
- **설명**: 사용자 입력이 인터프리터에 직접 전달되어 명령 실행
- **대응 MOIS**: IV-01(SQL), IV-03(XSS), IV-04(OS명령), IV-07(XQuery), IV-08(XPath), IV-09(LDAP), IV-11(HTTP분할), IV-12(정수오버플로), IV-14(버퍼), IV-15(포맷스트링)
- **대응 KISA**: WV-01(버퍼오버플로우), WV-02(포맷스트링), WV-03(LDAP), WV-04(OS명령), WV-05(SQL), WV-07(XPath), WV-11(XSS)
- **공공기관 주의**: 시큐어코딩 의무 항목 중 가장 빈번한 발견 유형

### A04:2021 Insecure Design (불안전한 설계)
- **설명**: 설계 단계에서의 보안 결함
- **대응 MOIS**: IV-05(파일업로드), IV-13(부적절한 입력값), TS-01(TOCTOU), TS-02(재귀), CE-01~04(코드오류), EC-04,05(Private배열)
- **대응 KISA**: WV-22(파일 업로드)
- **공공기관 주의**: 설계단계 보안설계 20개 항목 별도 검토 필요

### A05:2021 Security Misconfiguration (보안 설정 오류)
- **설명**: 보안 설정이 누락되거나 부적절하게 구성
- **대응 MOIS**: EH-01(에러노출), EH-02(에러대응), EH-03(예외처리), EC-02(디버그코드), EC-03(시스템정보노출)
- **대응 KISA**: WV-08(디렉터리 인덱싱), WV-09(정보 누출)
- **공공기관 주의**: 운영 환경 디버그 모드, 기본 에러 페이지, 서버 정보 노출 점검

### A06:2021 Vulnerable and Outdated Components (취약 컴포넌트)
- **설명**: 알려진 취약점이 있는 라이브러리/프레임워크 사용
- **대응 MOIS**: AM-02(취약한 API 사용)
- **공공기관 주의**: 공급망 보안 강화 추세, 외부 라이브러리 버전 관리 필수

### A07:2021 Identification and Authentication Failures (인증 실패)
- **설명**: 인증 및 세션 관리 결함
- **대응 MOIS**: SF-01(인증없는 중요기능), SF-07(하드코딩 비번), EC-01(세션정보노출), AM-01(DNS보안결정)
- **대응 KISA**: WV-12(약한 문자열 강도), WV-13(불충분한 인증), WV-14(취약한 패스워드 복구), WV-16(세션 예측), WV-18(불충분한 세션 만료), WV-19(세션 고정), WV-28(쿠키 변조)
- **공공기관 주의**: 로그인 실패 잠금, 세션 타임아웃, 비밀번호 복잡도 정책 필수

### A08:2021 Software and Data Integrity Failures (무결성 실패)
- **설명**: 무결성 검증 없이 소프트웨어 업데이트나 데이터 역직렬화
- **대응 MOIS**: CE-05(역직렬화)
- **공공기관 주의**: pickle, Java 직렬화 등 위험 역직렬화 사용 금지

### A09:2021 Security Logging and Monitoring Failures (로깅/모니터링 실패)
- **설명**: 보안 이벤트의 로깅/모니터링 부재
- **대응 MOIS**: 직접 대응 없음 (운영 영역)
- **공공기관 주의**: 침해사고 탐지를 위한 충분한 로깅 필수, 국정원 사이버보안 실태평가 항목

### A10:2021 Server-Side Request Forgery (SSRF)
- **설명**: 서버 측에서 사용자 제공 URL로 요청을 보내는 취약점
- **대응 MOIS**: 직접 대응 없음 (최신 가이드에서 별도 추가)
- **공공기관 주의**: 내부 네트워크 접근, 클라우드 메타데이터 탈취 위험
