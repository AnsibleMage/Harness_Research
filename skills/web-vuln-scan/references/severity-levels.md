# 심각도 분류 기준

## 개요
웹 취약점의 심각도를 4단계로 분류하는 기준. 공공기관 보안 감리에서 우선순위 결정에 활용.

## 심각도 등급

| 등급 | 코드 | 한국어 | 기준 | 예시 |
|------|------|--------|------|------|
| Critical | `critical` | 심각 | 즉시 악용 가능, 시스템 전체 영향 | SQL 인젝션, 원격 코드 실행, 인증 우회 |
| Major | `major` | 주요 | 추가 조건 필요하나 위험도 높음 | XSS, CSRF, 세션 고정, 정보 노출 |
| Minor | `minor` | 경미 | 직접 악용 어렵지만 공격 표면 노출 | 보안 헤더 누락, 약한 설정, 디버그 코드 |
| Info | `info` | 참고 | 자동 판단 불가, Best Practice | 수동 확인 필요 항목, 설정 권고 |

## 심각도 조정 규칙

| 조건 | 조정 | 예시 |
|------|------|------|
| 인증/결제/관리자 영역에서 발생 | 한 단계 상향 | Minor → Major |
| 내부 네트워크 전용 서비스 | 한 단계 하향 가능 | Major → Minor |
| 복합 취약점 (체인 가능) | 최고 등급으로 상향 | Minor + Minor = Critical |

## 종합 등급 산정 (보고서용)

| 등급 | 기준 |
|------|------|
| A (우수) | Critical 0건, Major 0건 |
| B (양호) | Critical 0건, Major 1~2건 |
| C (보통) | Critical 0건, Major 3건 이상 |
| D (미흡) | Critical 1~2건 |
| F (취약) | Critical 3건 이상 |
