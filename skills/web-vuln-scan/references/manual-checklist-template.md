# 수동 점검 체크리스트 템플릿

## 개요
Tier 1/2/3 자동 검사로 커버되지 않는 항목에 대한 수동 점검 체크리스트.
점검 담당자가 직접 확인해야 하는 항목을 정리.

## 사용 방법
아래 체크리스트를 복사하여 점검 보고서에 포함한다.
각 항목별로 양호/취약/해당없음/미확인 중 하나를 선택한다.

## 체크리스트

### 1. 서버 설정 보안 (Tier 1/2/3 미커버)
- [ ] 웹 서버(Apache/Nginx/IIS) 버전 정보 은닉 설정
- [ ] 불필요한 HTTP 메서드(TRACE, DELETE, PUT) 비활성화
- [ ] 디렉토리 리스팅 비활성화 (서버 설정 수준)
- [ ] 에러 페이지 커스터마이징 (기본 에러 페이지 사용 금지)
- [ ] 관리자 페이지 접근 IP 제한
- [ ] 불필요한 서비스/포트 비활성화

### 2. SSL/TLS 보안
- [ ] TLS 1.2 이상만 허용 (SSL 3.0, TLS 1.0/1.1 비활성화)
- [ ] 인증서 유효기간 확인
- [ ] 강력한 암호 스위트만 허용
- [ ] HSTS 헤더 설정 확인
- [ ] HTTP→HTTPS 리다이렉트 설정

### 3. 인증 및 세션 관리
- [ ] 로그인 실패 시 계정 잠금 정책 (5회 이상)
- [ ] 세션 타임아웃 설정 (30분 이내)
- [ ] 로그아웃 시 세션 완전 무효화
- [ ] 비밀번호 복잡도 정책 (8자 이상, 영/숫/특수)
- [ ] 중요 기능 접근 시 재인증
- [ ] 세션 ID 재발급 (로그인 성공 시)
- [ ] 취약한 패스워드 복구 절차 방지

### 4. 접근 제어
- [ ] 수평적 권한 상승 차단 (다른 사용자 데이터 접근)
- [ ] 수직적 권한 상승 차단 (관리자 기능 접근)
- [ ] URL 직접 접근 시 인가 검증
- [ ] 관리자 페이지 별도 경로 및 접근 제한
- [ ] 자동화 공격 방지 (CAPTCHA, Rate Limiting)

### 5. 데이터 보호
- [ ] 개인정보 마스킹 처리 (주민번호, 카드번호 등)
- [ ] 로그에 민감정보 기록 금지
- [ ] 개인정보 전송 시 암호화
- [ ] 쿠키에 민감정보 저장 금지
- [ ] 데이터 평문 전송 방지

### 6. 파일 처리
- [ ] 파일 업로드 시 확장자/MIME 타입/매직 바이트 검증
- [ ] 업로드 파일 실행 권한 제거
- [ ] 업로드 경로를 웹 루트 외부에 설정
- [ ] 파일 다운로드 시 경로 조작 방지
- [ ] 업로드 파일 크기 제한

### 7. 기타
- [ ] 소스코드 내 주석으로 민감정보 미노출
- [ ] 운영 환경에서 디버그 모드 비활성화
- [ ] 외부 라이브러리/프레임워크 최신 버전 확인
- [ ] CORS 설정에서 와일드카드(*) 사용 금지
- [ ] 프로세스 검증 누락 방지 (비즈니스 로직 우회)
- [ ] SSI(Server Side Include) 인젝션 방지
