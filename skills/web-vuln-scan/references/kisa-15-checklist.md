# KISA 웹 취약점 점검 항목 상세

## 개요
KISA 「주요정보통신기반시설 기술적 취약점 분석·평가 방법 상세가이드」의 웹(Web) 28개 점검 항목.
각 항목별 점검 설명, 위험도, 매핑 정보, Tier별 검사 방법, 판정 기준을 포함.

> 파일명은 하위 호환을 위해 kisa-15로 유지하나, 실제 28개 항목을 모두 포함.

## 항목 상세

### WV-05 SQL 인젝션
- **위험도**: 상
- **설명**: 사용자 입력이 SQL 쿼리에 직접 삽입되어 데이터베이스를 조작할 수 있는 취약점
- **OWASP**: A03:2021 Injection
- **MOIS**: MOIS-IV-01
- **Tier 1**: f-string/문자열 연결로 SQL 구성하는 패턴 탐지
- **Tier 2**: security-reviewer가 데이터 흐름 추적, parameterized query 사용 여부 확인
- **Tier 3**: `' OR 1=1--` 등 페이로드 전송 후 에러/응답 차이 분석
- **판정**: 양호(PreparedStatement/바인드 변수 사용) / 취약(문자열 연결 SQL)
- **수정**: Parameterized Query, ORM 사용, 입력값 화이트리스트 검증

### WV-11 크로스사이트 스크립팅(XSS)
- **위험도**: 상
- **설명**: 사용자 입력이 HTML에 이스케이프 없이 삽입되어 악성 스크립트가 실행되는 취약점
- **OWASP**: A03:2021 Injection
- **MOIS**: MOIS-IV-03
- **Tier 1**: innerHTML, document.write, echo 직접 출력 패턴 탐지
- **Tier 2**: security-reviewer가 출력 컨텍스트별 이스케이프 적용 여부 확인
- **Tier 3**: XSS 페이로드(`<script>alert(1)</script>`) 입력 후 DOM 반영 확인
- **판정**: 양호(자동 이스케이프 또는 DOMPurify) / 취약(미이스케이프 출력)
- **수정**: 출력 시 HTML 이스케이프, textContent 사용, CSP 헤더 설정

### WV-15 CSRF
- **위험도**: 상
- **설명**: 인증된 사용자의 의도하지 않은 요청을 위조하여 실행하는 취약점
- **OWASP**: A01:2021 BAC
- **MOIS**: MOIS-IV-10
- **Tier 1**: POST 폼에서 CSRF 토큰 hidden 필드 존재 여부 확인
- **Tier 2**: security-reviewer가 토큰 생성/검증 로직 분석
- **Tier 3**: CSRF 토큰 없이 폼 제출 시도 → 서버 응답 확인
- **판정**: 양호(CSRF 토큰 생성+검증) / 취약(토큰 부재 또는 미검증)
- **수정**: CSRF 토큰 프레임워크 기능 활성화, SameSite 쿠키 설정

### WV-09 정보 누출
- **위험도**: 중
- **설명**: 에러 메시지, 주석, 응답 헤더 등에서 시스템 내부 정보가 노출되는 취약점
- **OWASP**: A05:2021 Misconfig
- **MOIS**: MOIS-EH-01, MOIS-EC-03
- **Tier 1**: 소스코드 주석 내 비밀번호/키/내부IP 패턴, 에러 메시지 직접 반환 패턴
- **Tier 2**: security-reviewer가 에러 핸들링 전체 흐름 검토
- **Tier 3**: 의도적 에러 유발 → 스택 트레이스/버전 정보 노출 확인
- **판정**: 양호(커스텀 에러 페이지, 정보 미노출) / 취약(스택 트레이스 노출)
- **수정**: 커스텀 에러 페이지, 디버그 모드 비활성화, 주석 정리

### WV-27 데이터 평문 전송
- **위험도**: 중
- **설명**: 중요 데이터(개인정보, 인증정보)가 암호화 없이 전송되는 취약점
- **OWASP**: A02:2021 Crypto
- **MOIS**: MOIS-SF-06
- **Tier 1**: HTTP URL 사용, 암호화 없는 소켓 통신 패턴
- **Tier 2**: security-reviewer가 데이터 전송 경로의 암호화 적용 여부 확인
- **Tier 3**: HTTPS 리다이렉트 확인, HSTS 헤더 존재 확인
- **판정**: 양호(HTTPS 전용, HSTS 설정) / 취약(HTTP 통신 가능)
- **수정**: HTTPS 강제 리다이렉트, HSTS 헤더 설정, 인증서 관리

### WV-25 경로 추적
- **위험도**: 상
- **설명**: `../` 등을 이용하여 의도하지 않은 파일에 접근하는 취약점
- **OWASP**: A01:2021 BAC
- **MOIS**: MOIS-IV-02
- **Tier 1**: 파일 경로에 사용자 입력이 직접 연결되는 패턴
- **Tier 2**: security-reviewer가 경로 정규화(canonicalize) 적용 여부 확인
- **Tier 3**: `../../../etc/passwd` 요청 후 응답 확인
- **판정**: 양호(경로 정규화, 화이트리스트) / 취약(직접 경로 연결)
- **수정**: 경로 정규화, 기본 디렉토리 제한, 화이트리스트 검증
