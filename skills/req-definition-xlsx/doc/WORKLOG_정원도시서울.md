## 로그 구조 규칙

```
Session (세션 단위)
└── Prompt (사용자 프롬프트 단위)
    ├── 컨텍스트: 제공 자료, 참조 파일
    ├── 체인 선택: 어떤 체인을 왜 선택했는가
    └── Block (실행 블록 단위)
        ├── 도구/에이전트 호출
        ├── 입력 → 출력
        └── 판단 근거
```

**기록 원칙**:
- 모든 도구 호출을 빠짐없이 기록 (Read, Write, Bash, Agent 등)
- 에이전트는 subagent_type, model, 프롬프트 요약, 결과 요약 포함
- 병렬 호출은 `∥` 표기, 순차는 `→` 표기
- 실패/재시도가 있으면 사유와 해결책 기록
- 소요 시간은 에이전트 duration 기준
