## Chapter 1. 공통 헤더

> 모든 산출물 문서의 **상단 고정 영역**. 두 부분으로 구성된다.

### 1.1 Frontmatter (YAML)

```yaml
---
doc_id: "XXXXX"                    # 5자리 문서 번호 (폴더 체계와 동일)
title: "문서 제목"
project: "사업명"
phase: "착수 | 분석 | 설계 | 구현 | 시험 | 전개배포 | 종료"
version: "1.0.0"
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
author: "작성자명 (소속)"
reviewer: "검토자명 (소속)"
approver: "승인자명 (소속)"
status: draft | review | approved | final
classification: 일반 | 대외비 | 비밀
tags: [사업명, 단계, 산출물유형]
---
```

#### 필드 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `doc_id` | O | [[01_001_Folder_Structure_Methodology#3.2 5자리 숫자 계층 체계 설계|5자리 문서 번호]] — 전체 프로젝트에서 고유 |
| `title` | O | 산출물 공식 명칭 |
| `project` | O | 사업명 (예: "공원관리시스템 구축") |
| `phase` | O | 서울시 방법론 단계명 |
| `version` | O | SemVer (Major.Minor.Patch) |
| `created` | O | 최초 작성일 |
| `updated` | O | 최종 수정일 |
| `author` | O | 작성자 및 소속 |
| `reviewer` | △ | 검토자 (review 이후 필수) |
| `approver` | △ | 승인자 (approved 이후 필수) |
| `status` | O | 문서 상태 |
| `classification` | O | 보안 등급 |
| `tags` | △ | 옵시디언 태그 |

#### 문서 상태 흐름

```
draft → review → approved → final
  ↑       ↓
  └── rejected (사유 기록 후 draft로 회귀)
```

### 1.2 표지 블록

Frontmatter 바로 아래에 위치하는 **시각적 표지**.

```markdown
> **[사업명]**
> **[문서 제목]**
>
> | 항목 | 내용 |
> |------|------|
> | 문서번호 | XXXXX |
> | 버전 | v1.0.0 |
> | 작성일 | YYYY-MM-DD |
> | 작성자 | 홍길동 (개발팀) |
> | 검토자 | 김철수 (PM) |
> | 승인자 | 이영희 (발주처) |
> | 보안등급 | 일반 |
```

## Chapter 2. 몸통 작성 규칙

> 산출물 본문을 작성할 때 준수하는 규칙. 산출물마다 본문 구조는 다르지만, **서술 방식은 동일**하다.

### 2.1 서술 원칙

| 원칙 | 올바른 예 | 잘못된 예 |
|------|----------|----------|
| **구체적 서술** | "파일 크기 50MB 이하로 제한" | "적절한 크기로 제한" |
| **수치 포함** | "응답시간 3초 이내" | "빠른 응답시간" |
| **능동태 사용** | "시스템이 파일을 검증한다" | "파일이 검증된다" |
| **단문 우선** | 1문장 1내용 | 복문, 중문 남발 |
| **테이블 우선** | 비교/목록은 테이블로 | 장문의 나열형 서술 |

### 2.2 제목 계층 규칙

| 레벨 | 용도 | 예시 |
|------|------|------|
| `#` (H1) | 문서 제목 (1개만) | `# 사용자 요구사항 정의서` |
| `##` (H2) | 대분류 (Chapter) | `## 1. 기능 요구사항` |
| `###` (H3) | 중분류 (Section) | `### 1.1 사용자 관리` |
| `####` (H4) | 소분류 (Item) | `#### 1.1.1 로그인 기능` |

> [!warning] 제목 규칙
> - H1은 문서당 **1개만** 사용
> - H5 이하는 사용 금지 — 4뎁스 초과 시 구조 재설계
> - 제목에 번호를 붙여 정렬 순서를 명확히 한다

### 2.3 본문 서식

#### 텍스트 강조

| 서식 | 용도 | 예시 |
|------|------|------|
| **굵게** | 핵심 용어, 강조 | **필수 요구사항** |
| *기울임* | 참고 사항, 부연 | *단, 비기능 요구사항 제외* |
| `코드` | 시스템명, 파일명, 명령어 | `upload_service.py` |
| ~~취소선~~ | 삭제된 내용 (이력 추적용) | ~~기존 50MB 제한~~ → 500MB |

#### 목록

```markdown
# 순서 있는 목록 — 절차, 단계, 우선순위
1. 첫 번째 단계
2. 두 번째 단계

# 순서 없는 목록 — 나열, 특성, 조건
- 항목 A
- 항목 B
```

#### 테이블

```markdown
| 요구사항 ID | 요구사항명 | 우선순위 | 비고 |
|------------|----------|---------|------|
| REQ-001 | 대용량 파일 업로드 | 상 | 기술검토 필요 |
```

> [!tip] 테이블 사용 기준
> - 3개 이상의 항목을 비교할 때 → 테이블
> - 속성이 2개 이상인 목록 → 테이블
> - 단순 나열 → 불릿 리스트

#### 다이어그램

```markdown
# mermaid 블록으로 작성
graph TD
    A[시작] --> B{조건}
    B -->|Yes| C[처리]
    B -->|No| D[종료]
```

| 다이어그램 유형 | 용도 |
|---------------|------|
| `graph TD/LR` | 프로세스 흐름, 시스템 구조 |
| `sequenceDiagram` | 인터페이스 흐름, API 호출 순서 |
| `erDiagram` | 데이터 모델, 테이블 관계 |
| `gantt` | 일정 계획, WBS |

#### 콜아웃 (옵시디언)

| 유형 | 용도 |
|------|------|
| `> [!note]` | 참고 사항, 부연 설명 |
| `> [!warning]` | 주의 사항, 제약 조건 |
| `> [!important]` | 필수 준수 사항, 핵심 규칙 |
| `> [!tip]` | 권장 사항, 가이드 |
| `> [!danger]` | 위험 요소, 보안 이슈 |

### 2.4 교차 참조 규칙

산출물 간 참조 시 [[01_001_Folder_Structure_Methodology#3.2 5자리 숫자 계층 체계 설계|5자리 문서 번호]]를 사용한다.

```markdown
# 같은 프로젝트 내 산출물 참조
본 설계는 [[31001_사용자_요구사항_정의서#REQ-003|REQ-003 대용량 파일 업로드]]를 기반으로 한다.

# 기술검토서 참조
[[34001_기술검토서#네트워크 제약|네트워크 제약 분석]] 결과, 청크 분할 업로드 방식을 채택한다.
```

| 참조 형식 | 용도 |
|----------|------|
| `[[XXXXX_문서명#섹션\|표시명]]` | 특정 섹션 참조 (권장) |
| `[[XXXXX_문서명]]` | 문서 전체 참조 (지양) |
| `REQ-NNN` | 요구사항 ID 참조 |
| `TC-NNN` | 테스트 케이스 ID 참조 |

### 2.5 버전 표기

[[01_001_Folder_Structure_Methodology#3.6 서브넘버 규칙 — 버전/세션 분리|서브넘버 규칙]]에 따라 파일 단위 버전 관리:

| 변경 수준 | 버전 | 설명 |
|----------|------|------|
| 초기 작성 | `1.0.0` | 초안 완성 |
| 내용 추가/수정 | `1.x.0` | 섹션 추가, 요구사항 변경 |
| 오타/서식 | `1.0.x` | 단순 수정 |
| 전면 재작성 | `x.0.0` | 구조 변경 |

## Chapter 3. 공통 풋터

> 모든 산출물 문서의 **하단 고정 영역**. 세 부분으로 구성된다.

### 3.1 관련 문서 (Neural Map)

```markdown
## 관련 문서

### 직접 참조 (Direct Links)
- [[XXXXX_문서명#섹션|표시명]] — 이 문서의 근거/입력 산출물

### 역참조 (Backlinks)
- [[XXXXX_문서명#섹션|표시명]] — 이 문서를 입력으로 사용하는 산출물

### 관련 주제 (Topic Links)
- [[XXXXX_문서명#섹션|표시명]] — 동일 주제 다른 관점
```

> [!important] 최소 연결 규칙
> 모든 산출물은 최소 **2개 이상** 다른 산출물과 연결되어야 한다. 고립 문서(orphan) 금지.

### 3.2 릴리즈 노트 (Release Notes)

```markdown
---

## Release Notes

### v1.1.0 (2026-03-20)
- 요구사항 3건 추가

### v1.0.0 (2026-03-15)
- 초기 작성
> **프롬프트:** "원본 요청 내용"
```
